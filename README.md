# 網球王子 · 養成 App（yousei-app）

「生活記錄 → 角色養成」遊戲。把每天的生活記錄換成積分，投資到《網球王子》21 位角色的
15 項能力屬性上、學絕招、送禮物。純前端 SPA，可部署到 GitHub Pages（無後端），手機豎屏優先。

> 本實作對應三份規格：《app開發指令》《養成引擎設計文檔》《Sheet導出頁搭建指南》。
> **核心鐵律：只加不減、地板為 0、永不降階。**

---

## 快速開始

```bash
npm install
npm run dev       # 開發伺服器 http://localhost:5173
npm run build     # 產出 dist/（相對路徑，可直接丟 GitHub Pages）
npm run preview   # 預覽 build 結果
```

開啟後 → 右上角 **⚙️ 設定 → 角色導入** → 點「↻ 載入內建範例（21角色/101技能）」即可直接試玩；
或上傳自己的 `characters.csv` / `skills.csv`（格式見《Sheet導出頁搭建指南》）。

**每日流程**：⚙️ 設定 →「今日結算」算分入庫 →「分配中心」統一賦分（積分投屬性、技能點學絕招，
可同時操作多名角色、草稿可一鍵還原、當日點數不必用完）。主頁專注展示角色與圖文記憶。

## 架構（數據與程式分離）

代碼是「空殼引擎」，**不硬編任何角色/技能/成長數據**，全部從導入檔讀。

```
src/lib/
  constants.js   屬性鍵/分組/標籤、中文名→key 映射、戰隊配色、算分規則表（純常量）
  engine.js      純函數引擎：四板塊算分 / 技能點 / 心意幣 / 投資×1.5 / 升階 / 雙閘門
  parse.js       CSV(角色+技能) / JSON 解析 + 嚴格校驗（指出哪一行哪一欄）
  storage.js     IndexedDB(localForage)：模板 / 存檔 / 自動備份，分開存儲
  store.jsx      React Context：所有「會改存檔」的動作集中於此，保證持久化+備份
src/components/  UI：Home / 角色面板 / 三雷達圖 / 赋值 / 技能 / 禮物 / 設定四面板
```

**靜態模板**（StaticTemplate，導入後不變）與**動態存檔**（SaveState，每天在變）**分開存儲、分開備份**。
導入新模板不會清空養成進度。

## 與規格的兩處實作選擇

規格寫「圖表用 recharts、UI 用 Tailwind **或等價方案**」。本實作：

- **雷達圖**：手寫 SVG（`RadarChart.jsx`）取代 recharts。理由：要精確疊兩層（實線當前 /
  虛線目標）並把招牌屬性軸標星號+變色，自繪比套件更可控，且零額外依賴更穩。
- **樣式**：純 CSS（`index.css`，三戰隊配色用 CSS 變數即時切換）取代 Tailwind（規格允許等價方案）。

唯一執行期依賴是 `localforage`（IndexedDB 封裝）。

## 分階段交付狀態

- **Phase 1（最小可用閉環）** ✅ — 數據結構 + IndexedDB；CSV/JSON 導入；四板塊結算；三雷達圖 +
  赋值投積分 + 升階判定（Team/位置從數據讀）；導入/導出備份。
- **Phase 2** ✅ — 技能學習雙閘門 + 三態 UI；心意幣完整連續達成結算（過驗證用例）。
- **Phase 3** ✅ — 小貓商城 + 禮物贈送 + 甜蜜記憶；公共記憶區；自訂頭像導入；升階/學招/收禮提示動畫。

## 部署到 GitHub Pages

`vite.config.js` 已設 `base: './'`（相對路徑，任意子路徑都能跑）。

```bash
npm run build
# 把 dist/ 推到 gh-pages 分支，或用 GitHub Actions 部署 dist/
```

PWA：已含 `manifest.webmanifest` 與 iOS 全屏 meta，可「加到主屏」全屏運行。

## 數據導入格式（摘要）

- `characters.csv`：長格式，每行 = 一角色一屬性 ×6 階段。表頭
  `id,角色,战队,位置,成长类型,招牌属性,属性,阶1..阶6`。21 角色 × 15 屬性 = 315 行。
- `skills.csv`：每行一絕招。表頭 `id,charId,绝招,难度,必须阶段,技能点,属性门槛`；
  門檻用 `;` 分隔、每項 `中文屬性≥數值`。
- 容錯：BOM、CRLF/LF、全形數字、前後空白；校驗失敗精確報行/欄。
- 內建範例在 `public/sample/`，可一鍵載入。
